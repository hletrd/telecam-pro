## Phase 1 — Preview-output health and recorder independence

- ☐ **P1.1 / AGG-1 (High/High)** — Give each preview output generation an
  exactly-once failure owner. Surface attachment must return create/bind/init
  failure to `CameraEngine`; runtime make-current/draw/swap failure must carry
  the same Surface/generation identity so a stale output cannot invalidate a
  replacement. An owned failure publishes Not-Ready and chooses a bounded
  same-surface rebind or terminal renderer reset/reopen; repeated failure cannot
  loop forever. Acquire/update the real camera texture independently of preview
  availability, then render preview and encoder as sibling optional sinks. If
  recovery cannot preserve an active recorder, claim and terminate it through
  the existing ordered EGL-detach-before-codec-finalize path and clear the REC
  UI explicitly—never silently freeze its video tail. Add Android-free signal/
  branch tests for rejected attach, create/bind/init failure, preview swap
  failure before/after recorder readiness, stale replacement failure, bounded
  retry, encoder continuation or explicit termination, and exactly-once status.
  `GlPipeline.kt:151-230,500-598`; `CameraEngine.kt:499-560,620-630,
  2320-2385`; `CompletionDispatchTest.kt`; reconfiguration/ownership tests.
