- ☐ **P1.1 / AGG-1 (High/High)** — Make preview-output invalidation synchronous
  with a monotonic generation in `GlPipeline`. Every queued non-null/null output
  task must verify it still owns the latest generation before touching EGL;
  contain `eglCreateWindowSurface`/`makeCurrent` failure and converge to a
  detached output instead of escaping the HandlerThread. Add an Android-free
  generation test for bind A → invalidate/detach → drain stale A.
  `CameraScreen.kt:303-328`; `CameraEngine.kt:492-496,609-620`;
  `GlPipeline.kt:137-178,613-615`; `EglCore.kt:58-63`.

- ☐ **P1.2 / AGG-2 (High/High)** — Replace success-agnostic encoder completion
  with an exactly-once `Result<Unit>` dispatcher. A missing/dead GL owner or EGL
  exception must be an explicit failure; clearing an already absent owner remains
  successful. Queue attachment before recorder publication, retain its early
  result, and make UI REC readiness occur only after the identity-owned attach
  succeeds. A failed attach must claim and ordered-finalize the matching recorder,
  clean its pending MediaStore row, restore preview transfer, and surface failure.
  Test accepted success, rejected/throwing post, throwing task, exactly-once
  result, early completion, and failure/stop ownership races.
  `GlPipeline.kt:241-279,638-669`; `CameraEngine.kt:2031-2185`;
  `CameraViewModel.kt:301-315,1380-1405`; `CameraState.kt:585-608`;
  `CompletionDispatchTest.kt`.

- ☐ **P1.3 / AGG-5 (High/High)** — Introduce an irreversible terminal
  acquisition gate that serializes `gl.start` with engine release. Cold-start and
  GL-input callbacks must acquire the gate immediately before creating/enqueuing
  resources; `release()` closes it before the only final `gl.stop` and executor
  shutdown. A closed gate cannot return to started or dispatch Camera2 work.
  Add concurrency tests for close-before-run and close waiting for an in-flight
  acquisition, proving queued work cannot resurrect a generation.
  `CameraEngine.kt:492-552,2338-2373`; `GlPipeline.kt:115-135,492-567`.
