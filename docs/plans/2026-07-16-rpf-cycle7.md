## Phase 1 — EGL output ownership and recorder readiness

- ☑ **P1.1 / AGG-1 (High/High)** — Make EGL detach a checked ownership
  transition. On the GL owner thread, bind a distinct surviving output or make
  nothing current, then destroy the outgoing EGLSurface, then complete. Use this
  for encoder detach/replacement, preview clear/replacement, stale candidate
  cleanup, runtime output failure, and final GL release. Make
  `makeNothingCurrent`, surface destruction, presentation time, and buffer swap
  expose failure instead of discarding EGL status. Add an Android-free fake
  lifecycle seam proving `unbind -> destroy -> completion`, both preview/no-
  preview fallbacks, duplicate detach, and failing stages.
  `EglCore.kt:58-82`; `GlPipeline.kt:160-210,278-313,397-418,569-612`;
  `CompletionDispatchTest.kt` or a focused lifecycle suite.

- ☑ **P1.2 / AGG-2 (High/High)** — Replace allocation-ready with first-frame
  ready. Candidate attach must create, bind, and restore preview/nothing under
  one contained transaction; retain the attachment pending until the first real
  encoder frame presents and swaps successfully. Add a bounded no-frame timeout.
  A first-stage failure completes attach with failure; every later bind/draw/
  presentation/swap/restore failure signals the identity-owned recorder failure
  exactly once, detaches the bad output, and leaves preview rendering alive.
  Normal stop cancels a pending attach without creating a false failure for an
  already-claimed recorder. Share the recorder's early-failure publication path
  between codec drain and GL output failures. Test create/bind/restore failure,
  first/later swap failure, timeout, ready/failure races, and stop-before-first-
  frame; the UI tally begins only after the successful swap.
  `GlPipeline.kt:278-313,397-418`; `CameraEngine.kt:2182-2255`;
  `CameraViewModel.kt:301-308`; `CompletionDispatchTest.kt`.

## Phase 3 — Standards-defined SDR-to-HLG conversion

- ☑ **P3.1 / AGG-4 (High/High)** — Replace direct HLG OETF-on-SDR with the
  display-referred SDR-to-HLG path defined by current ITU guidance. Decode the
  display-referred BT.709-ish input with the BT.1886 2.4 approximation, convert
  linear BT.709 primaries to BT.2020, use an explicit normalized inverse-OOTF/
  reference-white mapping so 100% SDR maps to 75% HLG, then apply the BT.2100 HLG
  OETF. Share constants with an Android-free CPU reference and pin black,
  reference white, neutral grey, and RGB primary anchors; assert the shader
  contains the same ordered operations. Preserve the existing Main10/BT.2020/
  HLG container tags. Update source/architecture/product wording to call this an
  SDR-to-HLG mapping with no recovered highlights and no end-to-end 10-bit claim.
  Current references: ITU-R BT.2100-3 (02/2025) and BT.2408-9 (03/2026),
  especially BT.2408-9 §§5.1.1, 5.1.3, and 5.1.3.4.
  `Shaders.kt:13-18,52-65,97-107`; `ColorProfiles.kt:8-20`;
  `ColorTagsTest.kt`; new GL color conversion tests; `ARCHITECTURE.md:319-323,
