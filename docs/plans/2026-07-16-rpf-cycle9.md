## Phase 1 — Make preview and encoder frame ownership truthful

- ☑ **P1.1 / AGG-1 (High/High)** — Before every real
  `SurfaceTexture.updateTexImage`, choose and bind an identity-owned live output:
  preview first, otherwise the active encoder. Put make-current, texture update,
  and transform acquisition inside a checked boundary. A preview-owned failure
  must invalidate/recover that preview while preserving the encoder when safe;
  an encoder-owned or shared acquisition failure must converge through the
  existing encoder failure signal and ordered recorder teardown rather than
  escape the GL looper. Preserve the invariant that no codec window remains
  current between frames. Extract a small Android-free acquisition-owner policy
  and add tests for preview preference, encoder-only selection, no output,
  second encoder-only frame after make-nothing-current, and contained failure.
  `GlPipeline.kt:519-529,558-578,617-660`;
  `CompletionDispatchTest.kt` or a focused GL ownership suite.

- ☑ **P1.2 / AGG-2 (Medium/High)** — Split preview Attached from
  Renderable. `applyPreviewOutput` may install a pending health owner and create
  the camera input, but attachment completion must not signal UI Ready or reset
  retries. Only the first successful identity-owned preview swap transitions
  the signal to Ready; later runtime failure remains exactly once. Preserve
  recovery attempts across successful rebinds until that real swap, reset them
  only for a new surface generation or successful rendered frame, and stop
  scheduling after exhaustion. Test attach success without Ready,
  first-swap success, first-swap failure, stale replacement, and repeated
  bind-success/swap-failure exhaustion. `GlPipeline.kt:160-302,615-633`;
  `CameraEngine.kt:670-780,3165-3183`; `CompletionDispatchTest.kt`;
  `ReconfigurationGenerationTest.kt`.
